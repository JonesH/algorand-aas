Algorand Attestation Service (AAS)

Schema Registry + Attestation Writer with AI provenance

What you’ll see (≈4 minutes)

  • Canonicalize an AI inference run into a claim
  • Anchor a salted hash on Algorand via AAS
  • Verify attestation status and retrieve details
1. Setup 2. AI Canonicalization 3. Attest 4. Verify

1) Setup

Prereqs: Python 3.12+, uv, AlgoKit LocalNet (for full flow), LM Studio (optional for local model output)

# Install CLI
uv pip install -e .

# Set env for LocalNet (example)
export AAS_ALGOD_URL=http://localhost:4001
export AAS_ALGOD_TOKEN=aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa
export AAS_APP_ID=123456
export AAS_MNEMONIC="your 25 words ..."

# (Optional) Run included LM Studio example
scripts/lmstudio_example.sh

2) Canonicalize AI inference

Turn prompt + params + output into a deterministic ai.inference.v1 claim.

# Using the CLI
uv run aas ai canonicalize \
  --prompt examples/selfrun_gemma270m/prompt.txt \
  --params examples/selfrun_gemma270m/params.json \
  --output examples/selfrun_gemma270m/output.txt \
  --out examples/selfrun_gemma270m/claim.json

# Or run and canonicalize via LM Studio
scripts/lmstudio_run.sh \
  --prompt examples/selfrun_gemma270m/prompt.txt \
  --params examples/selfrun_gemma270m/params.json \
  --out-dir examples/selfrun_gemma270m \
  --attester demo-user

Outputs: claim.json with canonical fields and stable hash.

3) Create an attestation

Anchor a salted hash of the claim on Algorand via the AAS smart contract.

# Demo mode (placeholder signature)
uv run aas ai attest \
  --schema ai.inference.v1 \
  --claim examples/selfrun_gemma270m/claim.json

Production use: sign the canonical message off-chain with Ed25519 and pass real signature + attester key.

4) Verify and read back

Query attestation and check status.

# Lookup by Attestation ID (from the previous step)
uv run aas get <attestation_id_hex>

# (Optional) read the underlying box data for debugging
uv run python scripts/read_box.py $AAS_APP_ID <attestation_id_hex>

Result: Active attestation with subject, schema id, and CID/URI stored on-chain.
